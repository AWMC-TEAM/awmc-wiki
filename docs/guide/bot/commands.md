# AWMC QueryBot 指令调用说明

本页说明账号、上传、BREAK 与管理指令的调用方式。查歌、B50、猜歌、数据报告等完整
查询指令请见 [指令帮助](/guide/bot/advanced)。

::: warning 版本说明
账号功能已经合并到 QueryBot。旧 Koishi maiBot 的优先卡密、解绑卡、账号锁定、保护模式、
手工改分和收藏品维护指令未迁入当前版本，请勿继续按照旧教程调用。
:::

## 一、通用规则

- 指令前缀由平台配置决定；本文默认不写 `/`。
- 账号和上传流程会引用触发消息；敏感凭据直发因原消息会先撤回，群聊结果改用 `@用户` 标识。
- 交互流程通常支持 `取消`、`cancel`、`q` 或 `退出`。
- 二维码、官方二维码链接和 Token 都是敏感凭据，Bot 会优先尝试撤回。
- 外部调用会生成 `Ref_ID`；发生异常时请把它提供给管理员，不要发送 Token 或 UID。

## 二、协议与账号

| 指令 | 说明 |
|---|---|
| `用户协议` | 查看当前协议链接和确认方法 |
| `同意用户协议 <完整确认词>` | 确认协议；也可直接发送完整确认词 |
| `撤回用户协议` | 撤回当前版本的同意 |
| `mai账号` | 查看账号功能帮助 |
| `mai绑定` / `maibind` | 交互式绑定或认领舞萌账号 |
| `mai绑定 <二维码或链接>` | 一次提交并绑定 |
| `mai解绑` | 解绑舞萌账号，上传 Token 保留 |
| `mai状态` / `mymai` | 查看完整账号状态；缓存失效时重新询问二维码 |
| `maiping` | 检查 AWMC API 连接 |
| `maiqueue` | 查看公共网关队列状态 |
| `mai地图` | 查看游玩地区（服务支持时） |
| `mai查询opt <版本号>` | 查询 Mai2 选项文件 |

二维码输入支持：

```text
SGWCMAID...
https://wq.wahlap.net/qrcode/img/MAID....png
https://wq.wahlap.net/qrcode/req/MAID....html
```

凭据无效、过期、返回 500/全空或身份不一致时，Bot 最多重新询问 3 次。

## 三、查分器绑定与上传

| 指令 | 说明 |
|---|---|
| `mai绑定水鱼 <Token>` / `maibindfish <Token>` | 绑定水鱼 Token |
| `mai解绑水鱼` | 解绑水鱼 Token |
| `lxbind` | 绑定落雪 OAuth（推荐） |
| `lxunbind` | 解绑落雪 OAuth |
| `mai绑定落雪 <导入Token>` / `maibindlx <导入Token>` | 绑定落雪兼容 Token |
| `mai解绑落雪` | 解绑落雪兼容 Token |
| `maiu [二维码或链接]` | 上传水鱼 |
| `maiul [二维码或链接]` | 上传落雪 |
| `maiua [二维码或链接]` | 同时上传水鱼和落雪 |

三种上传共享每日首次成功免费额度。默认后续价格为水鱼 2、落雪 2、同时上传 3 BREAK；
失败、超时或取消均不扣费。

## 四、直接二维码与 PC

直接发送 `SGWCMAID...` 或受支持的官方链接，会自动完成 PC 同步，再按照绑定状态上传
水鱼、落雪或两边。未绑定查分器时只同步 PC。

| 指令 | 说明 |
|---|---|
| `更新pc数` | 使用缓存或最新二维码同步 PC |
| `我的pc数` | 查看个人 PC 统计 |
| `pc数 <歌曲>` | 查看指定谱面 PC |
| `pc50` | B50 按 PC 排序 |
| `pca50` | Rating B50 内按 PC 排序 |
| `游玩排行50` | 全库游玩最多的 50 个谱面 |
| `pc排行` | 所有已同步用户的 PC 总量排行 |

## 五、BREAK 与票券

| 指令 | 说明 |
|---|---|
| `AWMC签到` / `签到` | 每日签到 |
| `我的AWMC` | 查看 BREAK、连续签到、调用统计与最近流水 |
| `AWMC帮助` | 查看价格与奖励说明 |
| `转账BREAK @用户 数量` | 转账 |
| `BREAK抽奖 [1-10]` | 抽奖；默认每次消耗 2 BREAK |
| `发票` / `fp <2-6>` | 请求 2～6 倍票 |
| `mai查票` | 查询票券状态 |

发票每日首次成功免费，之后默认按倍率 × 2 BREAK 计费。外部服务真正成功后才会扣费。

## 六、管理员指令

以下指令仅限 Bot 超级管理员或插件管理员：

| 指令 | 说明 |
|---|---|
| `查询REF <Ref_ID>` | 查看脱敏后的完整处理链路 |
| `封禁用户 <用户> [原因]` | 封禁用户 |
| `解封用户 <用户>` | 解除封禁 |
| `封禁列表` | 查看生效中的封禁 |
| `管理面板` / `WebUI` | 查看 WebUI 地址与配置状态 |
| `查看AWMC <用户>` | 查看用户 BREAK 资料 |
| `设置BREAK <用户> <数量>` | 直接设置余额 |
| `增减BREAK <用户> <数量>` | 增减余额 |
| `BREAK配置 [键 值]` | 查看或修改 BREAK 默认参数 |
| `迁移Koishi 检查 <数据库>` | 只读预检 Koishi 账号迁移 |
| `迁移Koishi 确认 <数据库>` | 确认迁移账号、Token 和协议记录 |
| `存储状态` | 查看 SQLite/YAML/MySQL 存储状态 |
| `存储迁移 检查 <源> <目标>` | 预检数据迁移 |
| `存储迁移 确认 <源> <目标>` | 确认迁移 |
| `存储同步` | 立即同步当前配置的远端存储后端 |

WebUI 和管理接口不会返回二维码、Token 或完整街机 UID；管理员修改也会记录 `Ref_ID`。

## 七、交互示例

### 绑定并上传两边

```text
用户协议
mai绑定
mai绑定水鱼 <Token>
lxbind
maiua
```

### 二维码失效

```text
Bot：二维码缓存已失效，请发送最新二维码（最多 3 次）
用户：SGWCMAID...
Bot：验证成功并继续原操作
```

### 查询问题

```text
用户：mymai
Bot：引用该消息，返回账号状态与 Ref_ID
```
